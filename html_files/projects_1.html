TICKET SCANNER

A barcode scanning web application that dynamically processes CSV files and updates scanned records in an SQLite database, designed for event management with 300+ attendees.

Projects Link

Project Overview

This project is a robust web-based barcode scanning and event management system developed using Flask, designed to streamline the process of tracking and managing attendees at events with large crowds. The system is built to handle event registration and check-in processes by enabling users to upload CSV files containing attendee information. Upon uploading the CSV, the application dynamically processes the data and creates an SQLite database table with attendee records, including unique identifiers such as names and a barcode-matching "record_locator" field.

Key Features:

  • CSV Upload: Users can upload CSV files containing attendee information, which the system parses and stores in an SQLite database.
  • Dynamic Database Creation: The system processes the uploaded data and dynamically generates a database structure based on the CSV headers.
  • Real-time Barcode Scanning: Utilizing QuaggaJS, the system allows for seamless, real-time scanning of barcodes with live feedback.
  • Asynchronous Data Handling: AJAX requests are used to update the system in real-time, ensuring a responsive user experience.
  • Attendee Status Tracking: Each scanned record is updated in the database, and results are displayed to the user based on the status of the record: 'Found,' 'Not Found,' or 'Already Found.'
  • Recent Scans Monitoring: Displays the last five successfully scanned attendees for real-time event monitoring.
  • Responsive Interface: Built with Bootstrap, the application adapts to different screen sizes and device types, offering a smooth user experience.
